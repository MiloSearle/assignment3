What is JavaScript?

The Nature of JavaScript

JavaScript is a scripting language that allows web developers to implement features on web pages such as interactive maps, animated 2D or 3D graphics, etc. JavaScript is a functional and object-oriented language. Functional programming is features such as first-class functions and arrays and object-oriented programming are features such as objects which users can interact with such as buttons.

Objects and Methods in JavaScript

Objects in JavaScript are key/value pairs in which are properties and methods and have other JavaScript data types within them such as strings, numbers and Booleans. Objects originate from the parent Object constructor. This makes working with individual objects very easy and simple from the built-in methods that come with it such as Object.create(), Object.keys(), Object.values() and many more.

Event Handling in JavaScript

Event handling is JavaScript is used handle to when the state of an object is changing. This can be caused by user inputs, user actions and browser actions. Examples of this would be clicking on button or a mouse which can change the state of an object or go to a new page completely. You can code JavaScript so it executes when an event is detected such as a button being clicked.

Security Issues with JavaScript

JavaScript has extreme security issues which can be very damaging. Exploiting JavaScript can lead to hackers manipulating data, redirect sessions, modify and steal data. There are ways to prevent these security issues, the first way is to avoid using eval(), eval() executes a passed argument if it is JavaScript code or expression meaning that the hacker will be able to run any code or script that they want to. Another way to increase security is to use encryption such HTTPS which encrypts data which is transmitted between the client and the server meaning that it will be extremely hard to intercept.

Access Methods in JavaScript

Object methods can be accessed in JavaScript to be able to use it. In order to access a method, this line of code must be used - objectName.methodName(). This will then allow web developers to use the objects in areas of their webpage and then display them such as prompt boxes, alerts, confirmations etc.

The first issue that comes up is that people don't test on real devices first. Testing on devices is a very helpful way to make sure that your code does work on different browsers and if not, you can identify the different areas on the website which aren't working easily as it is your website and you know where the code is.

Another issue that comes up is that people do not validate their code for their website. Some browsers can handle code which isn't validated such as Google Chrome which can just auto-correct problems but browsers like Internet Explorer don't have an auto-correct feature and either the website will crash or the will just disregard the feature which isn't validated and not display it.

Vendor-specific functions is another issue that developers want to avoid as these can cause CSS to only work on certain browsers. Prefixes can cause it as well such as -webkit which is for Safari and Chrome, -ms which is for Internet Explorer, -moz which is for Mozilla Firefox and -o which is for Opera. These prefixes causes the function to only work in that browser which is something a developer doesn't want if they want it cross-browser compatible.

Importance of Website Security

Forms of Virus

Trojans

A Trojan virus is malware which disguises itself to be something else or perform something else but its true intent is to install malware onto the user's hardware, similar to the Trojan horse that snuck trojans into the city of Troy, which is why it is called a Trojan horse virus or Trojan virus. A way to protect your device from a Trojan virus is by installing an anti-virus against it, the anti-virus will scan frequently and effeciently to detect and alert you if a Trojan Virus is detected.

Worms

A worm virus is a program that can duplicate itself and spread through a network on its own and without the user needing to do something other downloading the malicous program itself. It is most commonly transmitted with file-sharing networks and emails. As Worms normally infiltrate from network services, a personal firewall would be a good solution to them as it will block external access to the network services and will prevent the worm from entering the system.

Hacking

Hacking is when an unauthorised user gains access into a system or network through a breach or cracking passwords and codes which allows access to the network. The best way to defend your system or network from hackers is with encryption as encryption encrypts data as it is moving through a network which makes the job very hard for a hacker to intercept as it will take much longer to access and find the access code to the encryption. If the hacker is identified accuractly, they can also be prosecuted with the Computer Misuse Act (1990) which defends personal data held by organisations from hackers. If a hacker goes against this act, the maximum penalty is up to 10 years in prison or an unlimited fine.

Identity Theft

Driver's License Fraud

Driver's license fraud is when someone has a driver's license given to them but under someone elses name so that any violations made on the road is on someone elses record and not theirs which could then end up on that person losing their license from doing nothing wrong. The main ways identity theft happens is from theft, phishing, cold calling and hacking. To prevent theft, you could have an alarm or CCTV around your home to identify who the theif is and also deter them if the alarm goes off. Phishing can be prevented by being cautious and knowing who you are being contacted by which can be done by checking their email address and if what they asking would actually be needed. Cold calling is when someone may pretend to be an organisation to gain your personal information, this can be prevented by just ignoring any unsolicited calls and assume every one has intention of gaining your personal information.

Financial Identity Theft

Financial identity theft is when someone uses your personal information to access things such as bank accounts. This type of fraud can take a very long time to recover from and can lead to debt and a bad credit score. The most common way this happens is through theft of someones wallet or purse, the best way to prevent this is to freeze all of your bank accounts and contact your bank immediately to fix the problem.

Child Identity Theft

Child identity theft is normally committed by a relative or a close family friend who takes out loans and credit cards in the child's name which the child will not notice as they don't need monitor their credit score and reports and will not notice until they come of age. Similar to financial identity fraud, this can take a long time to sort out and can come with some long term issues.

Change of Address Fraud

Change of address fraud is when someone changes your mailing address to divert information such as bank details and credit card details towards them which they can then steal and use for personal gain. This would be done through either phishing or cold calling so ignore out of place emails and ignore unsolicited calls.

Employment Identity Theft

Criminals, illegal immigrants and the unemployed can pretend to be you and gain employment through this as they are using your history in jobs and society and not theirs. This will most likely happen from theft as it requires identification and will be a face-to-face interview so in order to prevent this, home security needs to be put into place in order to deter theives from your home.

Data Security

The legislation that prosecutes organisations if a data breach occurs and is being shared is The Data Protection Act which controls how customer and personal data is used within an organisation and government bodies. A real world example of this being breached is when NHS files got into the wrong hands and a data breach happened for patients where their personal information was revealed which meant the NHS had to pay a lot of money as a consequence.

Methods for Protecting Data

One method for protecting data would be encryption. If data is encrypted, then it will be very hard to be accessed by hackers as the information to access the data will be disguised and hard to identify making it very annoying and will cause them to eventually give up if they quit easily but some hackers will be quite dedicated and may continue to try for a very long time which means that they may at some point eventually access the data but other security measures can be put into place to see if someone is attempting to access the data and they can then let the authorities take over after this point.

Another method is using access controls and firewalls. Access controls will add a layer of protection from the data as it acts a locked door that only one person has the key to but it can still be located and used to access the network by a hacker which is where firewalls will take place to monitor and manage access to the main or core data network from the Internet and portable devices such as phones and laptops.
